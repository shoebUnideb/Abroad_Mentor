The 5 Phases


Phase 1 — JWT Migration (hardest, do first, stay on one server)
Install djangorestframework-simplejwt
Rewrite LoginView to return {access, refresh, user} instead of setting a session cookie
Add LogoutView that blacklists the refresh token
Build JWTAuthMiddleware to replace AuthMiddlewareStack for WebSockets — passes token via ?token= query param
Rewrite apiClient.ts entirely: Authorization: Bearer header, auto-refresh on 401, no more CSRF
AuthContext restores user by decoding JWT from localStorage — instant, no network call on page load


Phase 2 — Frontend Split
Extract into public-frontend/ and internal-frontend/ — two independent Vite projects
Add AuthCallbackPage to internal-frontend to receive tokens via URL fragment (#access=TOKEN) — this is the SSO handoff between origins


Phase 3 — Backend Split
Three Django projects: auth-service, public-portal, internal-portal
Key pattern: public-portal and internal-portal each connect to both their own DB and the shared auth_db via a DATABASE_ROUTER — no model changes needed, FKs still work


Phase 4 — RS256 Asymmetric Keys
One openssl genrsa command generates the key pair
Only auth-service holds the private key — the others only get the public key and can verify but never issue tokens
This is the security model — even if a portal server is compromised, it cannot forge identities


Phase 5 — Deployment
Three Nginx configs, three Gunicorn/Daphne processes, three .env files
